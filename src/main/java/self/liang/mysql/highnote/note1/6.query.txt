    1）查询为什么会慢
    2）慢查询基础：优化数据访问：
        基本的原因是访问的数据太多。通过两个步骤分析总时有效的：
            1.确认应用程序是否在检索大量超过需要的数据，这通常意味着访问了太多的行，但有时候也可能是因为访问了太多列。
            2.确认mysql服务层是否在分析大量超过需要的数据行。
        1.是否向数据库请求了不需要的数据：
            典型案例：
                1.查询不需要的记录。
                    select查询大量的结果，然后获取前面的N行后关闭。最简单的解决方法是用limit
                2.多表关联返回全部列：
                3.总时取出全部列
                    select * 禁止
                4.重复查询相同的数据
